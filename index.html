Protocols CED


1Phisical layer
The physical layer refers to the physical communication medium and the technologies for transmitting data over that medium. At its core, data communication is the transfer of digital and electronic signals over various physical channels, such as fiber optic cables, copper cabling, and air

2Data link
The data link layer refers to the technologies used to connect two machines over a network where the physical layer already exists. Manages data frames, which are digital signals encapsulated in data packets

3Grid
La capa de red se ocupa de conceptos como el enrutamiento, el reenvío y el direccionamiento a través de una red dispersa o de múltiples redes conectadas de nodos o máquinas

4Tranport
The main purpose of the transport layer is to ensure that data packets arrive in the correct order, without loss or errors, or that they can be retrieved smoothly if necessary. Flow control, along with error control, is often a goal at the transport layer

5Session
The session layer is responsible for network coordination between two independent applications in a session. A session manages the start and end of synchronization and connection conflicts for an application one-to-one.

6Presntation
The presentation layer is primarily concerned with the syntax of the data itself for applications to send and consume.

7Applicationn
The application layer refers to the specific type of application itself and its standardized communication methods. For example, browsers can communicate using Hypertext Transfer Protocol (HTTPS) and email and HTTP clients can communicate using POP3 (Post Office Protocol version 3) and SMTP (Simple Mail Transfer Protocol).
